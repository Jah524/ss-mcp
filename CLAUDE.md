# CLAUDE.md

このファイルは Claude Code がこのリポジトリで作業する際のガイドラインです。

## コードレビュー

コードの変更を行った後は、必ず `code-review` MCP ツールを使用してレビューを実施してください。

### レビュー手順

1. コード変更後、コミット前に `review_git_diff` ツールを実行：
   - `repo_root`: このリポジトリのパス
   - `diff_source`: `"staged"`（ステージ済み）または `"working"`（未ステージ）

2. レビュー結果の指摘事項を確認し、必要に応じて修正を行う

3. 修正が完了したら再度レビューを実行し、問題がないことを確認

4. 問題がなければコミットを実行

### レビュー観点

デフォルトでは以下の観点でレビューされます：
- バグ
- エッジケース
- セキュリティ
- パフォーマンス
- 可読性

特定の観点に絞りたい場合は `focus` パラメータで指定できます。
